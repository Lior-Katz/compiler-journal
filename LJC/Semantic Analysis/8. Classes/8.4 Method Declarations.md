# [8.4.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.4) Method Declarations
##### [8.4.1.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.4.1) Formal Parameters
- [ ] Every declaration of a formal parameter of a method or constructor must include an _Identifier_, otherwise a compile-time error occurs.
- [ ] It is a compile-time error for a method or constructor to declare two formal parameters with the same name.
- [ ] If the formal parameter is not a variable arity parameter, then the declared type is denoted by _UnannType_ if no bracket pairs appear in _UnannType_ and _VariableDeclaratorId_, and specified by [§10.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-10.html#jls-10.2 "10.2. Array Variables") otherwise.
###### modifiers
- [ ] only modifiers are `final` and annotations
- [ ] It is a compile-time error if `final` appears more than once as a modifier for a formal parameter declaration.
###### variable arity parameters
- [ ] At most one variable arity parameter is permitted
- [ ] It is a compile-time error if a variable arity parameter appears anywhere in the list of parameter specifiers except the last position.
- [ ] If the formal parameter is a variable arity parameter, then the declared type is an array type specified by [§10.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-10.html#jls-10.2 "10.2. Array Variables").
- [ ] If the declared type of a variable arity parameter has a non-reifiable element type ([§4.7](https://docs.oracle.com/javase/specs/jls/se26/html/jls-4.html#jls-4.7 "4.7. Reifiable Types")), then a compile-time unchecked warning occurs for the declaration of the variable arity method, unless the method is annotated with `@SafeVarargs` ([§9.6.4.7](https://docs.oracle.com/javase/specs/jls/se26/html/jls-9.html#jls-9.6.4.7 "9.6.4.7. @SafeVarargs")) or the warning is suppressed by `@SuppressWarnings` ([§9.6.4.5](https://docs.oracle.com/javase/specs/jls/se26/html/jls-9.html#jls-9.6.4.5 "9.6.4.5. @SuppressWarnings")).
