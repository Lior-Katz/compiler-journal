# [15.20.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.20) Relational Operators
- [x] The type of a relational expression is always `boolean`.
##### [15.20.1.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.20.1) Numerical Comparison Operators `<`, `<=`, `>`, and `>=`
- [ ] The type of each of the operands of a numerical comparison operator must be a type that is convertible ([§5.1.8](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.1.8 "5.1.8. Unboxing Conversion")) to a primitive numeric type, or a compile-time error occurs.
	- [x] primitive
	- [ ] unboxing
##### [15.20.2.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.20.2) The `instanceof` Operator
If the operand to the right of the `instanceof` keyword is a _ReferenceType_, then the `instanceof` keyword is the _type comparison operator_.
- [ ] The type of the expression _RelationalExpression_ must be a reference type or the null type, or a compile-time error occurs.
- [ ] The _RelationalExpression_ must be checked cast compatible with the _ReferenceType_ ([§5.5](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.5 "5.5. Casting Contexts")), or a compile-time error occurs.

If the operand to the right of the `instanceof` keyword is a _Pattern_, then the `instanceof` keyword is the _pattern match operator_.
- [ ] The type of the expression _RelationalExpression_ must be a reference type or the null type, or a compile-time error occurs.
- [ ] The _Pattern_ must be applicable at the type of the expression _RelationalExpression_ ([§14.30.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.30.3 "14.30.3. Properties of Patterns")), or a compile-time error occurs.
