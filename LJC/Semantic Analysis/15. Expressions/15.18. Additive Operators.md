# [15.18.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.18) Additive Operators
- [ ] If the type of either operand of a `+` operator is `String`, then the operation is string concatenation.
- [ ] Otherwise, the type of each of the operands of the `+` operator must be a type that is convertible ([§5.1.8](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.1.8 "5.1.8. Unboxing Conversion")) to a primitive numeric type, or a compile-time error occurs.
	- [x] primitive
	- [ ] boxed
- [ ] In every case, the type of each of the operands of the binary `-` operator must be a type that is convertible ([§5.1.8](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.1.8 "5.1.8. Unboxing Conversion")) to a primitive numeric type, or a compile-time error occurs.
	- [x] primitive
	- [ ] boxed
##### [15.18.1.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.18.1) String Concatenation Operator `+`
- [ ] The result of string concatenation is a reference to a `String` object that is the concatenation of the two operand strings.
##### [15.18.2.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.18.2) Additive Operators (`+` and `-`) for Numeric Types
- [ ] The type of an additive expression on numeric operands is the promoted type of its operands.
