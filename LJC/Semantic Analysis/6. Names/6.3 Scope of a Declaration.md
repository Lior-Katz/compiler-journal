# [6.3.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-6.html#jls-6.3) Scope of a Declaration
- [ ] The _scope_ of a declaration is the region of the program within which the entity declared by the declaration can be referred to using a simple name, provided it is not shadowed ([§6.4.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-6.html#jls-6.4.1 "6.4.1. Shadowing")).

- [ ] A declaration is said to be _in scope_ at a particular point in a program if and only if the declaration's scope includes that point.

- [ ] The scope of the declaration of an observable top level package ([§7.4.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.4.3 "7.4.3. Package Observability and Visibility")) is all observable compilation units associated with modules to which the package is uniquely visible ([§7.4.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.4.3 "7.4.3. Package Observability and Visibility")).

- [ ] The declaration of a package that is not observable is never in scope.

- [ ] The declaration of a subpackage is never in scope.

- [ ] The package `java` is always in scope.

- [ ] The scope of a class or interface imported by a single-type-import declaration ([§7.5.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.5.1 "7.5.1. Single-Type-Import Declarations")), a type-import-on-demand declaration ([§7.5.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.5.2 "7.5.2. Type-Import-on-Demand Declarations")), or a single-module-import declaration ([§7.5.5](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.5.5 "7.5.5. Single-Module-Import Declarations")) is the module declaration ([§7.7](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.7 "7.7. Module Declarations")) and all the class and interface declarations ([§8.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.1 "8.1. Class Declarations"), [§9.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-9.html#jls-9.1 "9.1. Interface Declarations")) of the compilation unit in which the `import` declaration appears, as well as any annotations on the module declaration or package declaration of the compilation unit.

- [ ] The scope of a member imported by a single-static-import declaration ([§7.5.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.5.3 "7.5.3. Single-Static-Import Declarations")) or a static-import-on-demand declaration ([§7.5.4](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.5.4 "7.5.4. Static-Import-on-Demand Declarations")) is the module declaration and all the class and interface declarations of the compilation unit in which the `import` declaration appears, as well as any annotations on the module declaration or package declaration of the compilation unit.

- [ ] The scope of a top level class or interface ([§7.6](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.6 "7.6. Top Level Class and Interface Declarations")) declared in an ordinary compilation unit is all the class and interface declarations in the package in which the top level class or interface is declared.

- [ ] The scope of the implicit declaration of a top level class in a compact compilation unit ([§7.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-7.html#jls-7.3 "7.3. Compilation Units")) is empty; the class is never in scope.

- [ ] The scope of a declaration of a member `m` declared in or inherited by a class or interface C ([§8.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.2 "8.2. Class Members"), [§9.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-9.html#jls-9.2 "9.2. Interface Members")) is the entire body of C, including any nested class or interface declarations. 

- [ ] If C is a record class, then the scope of `m` additionally includes the header of the record declaration of C.

- [ ] The scope of a formal parameter of a method ([§8.4.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.4.1 "8.4.1. Formal Parameters")), constructor ([§8.8.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.8.1 "8.8.1. Formal Parameters")), or lambda expression ([§15.27](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.27 "15.27. Lambda Expressions")) is the entire body of the method, constructor, or lambda expression.

- [ ] The scope of a class's type parameter ([§8.1.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.1.2 "8.1.2. Generic Classes and Type Parameters")) is the type parameter section of the class declaration, and the type parameter section of any superclass type or superinterface type of the class declaration, and the class body. If the class is a record class ([§8.10](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.10 "8.10. Record Classes")), then the scope of the type parameter additionally includes the header of the record declaration ([§8.10.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.10.1 "8.10.1. Record Components")).

- [ ] The scope of an interface's type parameter ([§9.1.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-9.html#jls-9.1.2 "9.1.2. Generic Interfaces and Type Parameters")) is the type parameter section of the interface declaration, and the type parameter section of any superinterface type of the interface declaration, and the interface body.

- [ ] The scope of a method's type parameter ([§8.4.4](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.4.4 "8.4.4. Generic Methods")) is the entire declaration of the method, including the type parameter section, but excluding the method modifiers.

- [ ] The scope of a constructor's type parameter ([§8.8.4](https://docs.oracle.com/javase/specs/jls/se26/html/jls-8.html#jls-8.8.4 "8.8.4. Generic Constructors")) is the entire declaration of the constructor, including the type parameter section, but excluding the constructor modifiers.

- [ ] The scope of a local class or interface declaration immediately enclosed by a block ([§14.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.2 "14.2. Blocks")) is the rest of the immediately enclosing block, including the local class or interface declaration itself.

- [ ] The scope of a local class or interface declaration immediately enclosed by a switch block statement group ([§14.11](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.11 "14.11. The switch Statement")) is the rest of the immediately enclosing switch block statement group, including the local class or interface declaration itself.

- [ ] The scope of a local variable declared in a block by a local variable declaration statement ([§14.4.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.4.2 "14.4.2. Local Variable Declaration Statements")) is the rest of the block, starting with the declaration's own initializer and including any further declarators to the right in the local variable declaration statement.

- [ ] The scope of a local variable declared in the _ForInit_ part of a basic `for` statement ([§14.14.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.14.1 "14.14.1. The basic for Statement")) includes all of the following:
	- Its own initializer
	- Any further declarators to the right in the _ForInit_ part of the `for` statement
	- The _Expression_ and _ForUpdate_ parts of the `for` statement
	- The contained _Statement_

- [ ] The scope of a local variable declared in the header of an enhanced `for` statement ([§14.14.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.14.2 "14.14.2. The enhanced for statement")) is the contained _Statement_.

- [ ] The scope of a local variable declared in the resource specification of a `try`-with-resources statement ([§14.20.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.20.3 "14.20.3. try-with-resources")) is from the declaration rightward over the remainder of the resource specification and the entire `try` block associated with the `try`-with-resources statement.

- [ ] The scope of a parameter of an exception handler that is declared in a `catch` clause of a `try` statement ([§14.20](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.20 "14.20. The try statement")) is the entire block associated with the `catch`.

- [ ] The scope of a pattern variable declaration (that is, a local variable declared by a pattern) is the part of the program that might be executed after the matching of a value against the pattern has succeeded ([§14.30.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.30.2 "14.30.2. Pattern Matching")). It is determined by considering the program points where the pattern variable is _definitely matched_ in a region beginning with the pattern that declares the pattern variable.
