# [14.4.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.4) Local Variable Declarations
###### Modifiers
- [ ] only modifiers are `final` and annotations
- [ ] It is a compile-time error if `final` appears more than once as a modifier for a formal parameter declaration.

It is a compile-time error if a local variable declaration that (i) does not include an _Identifier_ and (ii) does not have an initializer, appears in any of the following locations:
- [ ] a local variable declaration statement in a block ([§14.4.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.4.2 "14.4.2. Local Variable Declaration Statements"))
- [ ] the header of a basic `for` statement ([§14.14.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.14.1 "14.14.1. The basic for Statement"))

It is a compile-time error if the _LocalVariableType_ is `var` and any of the following are true:
- [ ] More than one _VariableDeclarator_ is listed.
- [ ] The _VariableDeclaratorId_ has one or more bracket pairs.
- [ ] The _VariableDeclarator_ lacks an initializer.
- [ ] The initializer of the _VariableDeclarator_ is an _ArrayInitializer_.
- [ ] The initializer of the _VariableDeclarator_ contains a reference to the variable.
##### [14.4.1.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.4.1) Local Variable Declarators and Types
- [ ] It is a compile-time error if the _LocalVariableType_ is `var`, and the type of the initializer is the null type
- [ ] an error occurs if the initializer is a lambda expression ([§15.27](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.27 "15.27. Lambda Expressions")) or a method reference expression ([§15.13](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.13 "15.13. Method Reference Expressions"))

##### [14.4.2.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-14.html#jls-14.4.2) Local Variable Declaration Statements
- [ ] If a declarator does not have an initializer, then every reference to the variable must be preceded by execution of an assignment to the variable, or a compile-time error occurs by the rules of [§16 (_Definite Assignment_)](https://docs.oracle.com/javase/specs/jls/se26/html/jls-16.html "Chapter 16. Definite Assignment").
