# [5.6.](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.6) Numeric Contexts

##### Numeric context
An expression appears in a _numeric arithmetic context_ if the expression is one of the following:
- [x] The operand of a unary plus operator `+`, unary minus operator `-`, or bitwise complement operator `~` ([§15.15.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.15.3 "15.15.3. Unary Plus Operator +"), [§15.15.4](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.15.4 "15.15.4. Unary Minus Operator -"), [§15.15.5](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.15.5 "15.15.5. Bitwise Complement Operator ~"))
- [x] An operand of a multiplicative operator `*`, `/`, or `%` ([§15.17](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.17 "15.17. Multiplicative Operators"))
- [ ] An operand of an addition or subtraction operator for numeric types `+` or `-` ([§15.18.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.18.2 "15.18.2. Additive Operators (+ and -) for Numeric Types"))
- [ ] An operand of a shift operator `<<`, `>>`, or `>>>` ([§15.19](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.19 "15.19. Shift Operators")). Operands of these shift operators are treated separately rather than as a group. A `long` shift distance (right operand) does not promote the value being shifted (left operand) to `long`.
- [ ] An operand of a numerical comparison operator `<`, `<=`, `>`, or `>=` ([§15.20.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.20.1 "15.20.1. Numerical Comparison Operators <, <=, >, and >="))
- [ ] An operand of a numerical equality operator `==` or `!=` ([§15.21.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.21.1 "15.21.1. Numerical Equality Operators == and !="))    
- [ ] An operand of an integer bitwise operator `&`, `^`, or `|` ([§15.22.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.22.1 "15.22.1. Integer Bitwise Operators &, ^, and |"))

An expression appears in a _numeric array context_ if the expression is one of the following:
- [ ] A dimension expression in an array creation expression ([§15.10.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.10.1 "15.10.1. Array Creation Expressions"))
- [ ] The index expression in an array access expression ([§15.10.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.10.3 "15.10.3. Array Access Expressions"))

An expression appears in a _numeric choice context_ if the expression is one of the following:
- [x] The second or third operand of a numeric conditional expression ([§15.25.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.25.2 "15.25.2. Numeric Conditional Expressions"))
- [ ] A result expression of a standalone `switch` expression ([§15.28.1](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.28.1 "15.28.1. The Switch Block of a switch Expression")) where all the result expressions are convertible to a numeric type

##### Numeric Promotion:
1. [x] If any expression is of a reference type, it is subjected to unboxing conversion ([§5.1.8](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.1.8 "5.1.8. Unboxing Conversion")).
2. [ ] Next, widening primitive conversion ([§5.1.2](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.1.2 "5.1.2. Widening Primitive Conversion")) and narrowing primitive conversion ([§5.1.3](https://docs.oracle.com/javase/specs/jls/se26/html/jls-5.html#jls-5.1.3 "5.1.3. Narrowing Primitive Conversion")) are applied to some expressions, according to the following rules:

    - [x] If any expression is of type `double`, then the promoted type is `double`, and other expressions that are not of type `double` undergo widening primitive conversion to `double`.
  
    - [x] Otherwise, if any expression is of type `float`, then the promoted type is `float`, and other expressions that are not of type `float` undergo widening primitive conversion to `float`.
  
    - [x] Otherwise, if any expression is of type `long`, then the promoted type is `long`, and other expressions that are not of type `long` undergo widening primitive conversion to `long`.

     Otherwise, none of the expressions are of type `double`, `float`, or `long`. In this case, the kind of context determines how the promoted type is chosen.
    
    - [x] In a numeric arithmetic context or a numeric array context, the promoted type is `int`, and any expressions that are not of type `int` undergo widening primitive conversion to `int`.
    - [ ] In a numeric choice context, the following rules apply:
	    - [x] If any expression is of type `int` and is not a constant expression ([§15.29](https://docs.oracle.com/javase/specs/jls/se26/html/jls-15.html#jls-15.29 "15.29. Constant Expressions")), then the promoted type is `int`, and other expressions that are not of type `int` undergo widening primitive conversion to `int`.
		    - [ ] constants
		    - [x] not constants (currently everything)
	
        - [x] Otherwise, if any expression is of type `short`, and every other expression is either of type `short` or of type `byte` or a constant expression of type `int` with a value that is representable in the type `short`, then the promoted type is `short`, and the `byte` expressions undergo widening primitive conversion to `short`, and the `int` expressions undergo narrowing primitive conversion to `short`.
	        - [ ] constant ints
      
        - [x] Otherwise, if any expression is of type `byte`, and every other expression is either of type `byte` or a constant expression of type `int` with a value that is representable in the type `byte`, then the promoted type is `byte`, and the `int` expressions undergo narrowing primitive conversion to `byte`
	        - [ ] constant ints
	
        - [x] Otherwise, if any expression is of type `char`, and every other expression is either of type `char` or a constant expression of type `int` with a value that is representable in the type `char`, then the promoted type is `char`, and the `int` expressions undergo narrowing primitive conversion to `char`.
	
        - [x] Otherwise, the promoted type is `int`, and all the expressions that are not of type `int` undergo widening primitive conversion to `int`.
